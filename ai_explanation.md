本プロンプトの意図を正確に理解いたしました。プロの金融クオンツ・エンジニアとして、ご提示いただいた「グランビル法則スクリーナー」の誤判定事例を詳細に分析し、判定ロジックと期待度スコアの最適化を行います。

ユーザーからの貴重なフィードバックに基づき、より「人間の視覚的判断」に近づけるため、主に以下の改善を実施します。

### 1. 各誤判定事例に対する分析と改善提案

#### 📁 事例：1905.T (テノックス) - 2026-07-04 (中期(25/75))
- **現在の出力:** 買い3
- **ユーザーのフィードバック:** 「75日線接近以前に75日線の上昇を伴う上放れの株価上昇が起こっていない。判定するとしても上昇トレンド中の買い2の局面と考えられる。」
- **分析:**
    *   当時の75日線乖離率が3.10%と、長期線からやや離れた位置での「押し目反発」判定。
    *   75日線の当日変化率は過去半年平均より鈍化しており、強い上昇トレンドとは言い難い。
    *   買い3の条件 `is_close_to_ma = 0.0 < diff_rate <= 3.5` が、このケースでは「押し目」としては緩すぎると判断された可能性があります。また、本来の押し目買いは短期線が長期線の上または近くにあることが望ましいです。
- **改善提案:**
    1.  **買い3の「乖離率しきい値」を厳格化:** `0.0 < diff_rate <= 3.5` を `0.0 < diff_rate <= 2.0` に修正します。これにより、より長期線に近い位置での反発のみを「押し目」と判断します。
    2.  **買い3の「長期線の安定的な上昇」条件を強化:** `is_long_ma_rising_strong` を、直近15日間の上昇に加え、直近5日間も上昇しており、かつ本日も前日より上昇していることを条件とすることで、より持続的で明確な上昇トレンドであることを確認します。
    3.  **買い3に「短期線（25日線）の位置と傾き」を追加:** 短期線が長期線の上に位置し、かつ短期線自体が下向きになっていないこと（`short_ma_today > long_ma_today` かつ `short_ma_slope_3d >= 0`）を条件に追加します。これにより、下降トレンド中の押し目ではないことを明確にします。

#### 📁 事例：1982.T (日比谷総合設備) - 2026-07-04 (中期(25/75))
- **現在の出力:** 買い3
- **ユーザーのフィードバック:** 「上昇トレンド中のレンジ相場になっており、判定理由の「大きく上昇した」の部分に当てはまらない。当てはまるとすれば日足で5月13日の急騰だが、1日の急騰をもって上昇トレンドとは言えないので、条件設定か判定理由のどちらかに不備がある。」
- **分析:**
    *   当時の75日線乖離率が3.20%と、やはり長期線からやや離れた位置での判定。
    *   75日線の当日変化率が過去半年平均よりかなり鈍化しており、ユーザーの「レンジ相場」という認識と一致します。現在の `long_ma_slope_15d > 0` だけでは、一時的なレンジや鈍化を十分に排除できていませんでした。
- **改善提案:**
    *   上記のテノックス事例と同様、買い3の「乖離率しきい値」と「長期線の安定的な上昇」条件、および「短期線（25日線）の位置と傾き」を強化することで、レンジ相場やトレンド鈍化中の銘柄が買い3として誤判定されることを防ぎます。

#### 📁 事例：4119.T (日本ピグメントホールディングス) - 2026-07-06 (中期(25/75))
- **現在の出力:** 買い2
- **ユーザーのフィードバック:** 「長期にわたって75日線と共に上昇し、75日線を1度下抜けたあと反転気配があるのでどちらかというと買い3」
- **分析:**
    *   当時の75日線乖離率が3.57%、25日線乖離率が6.94%と、長期線だけでなく短期線からも株価が大きく乖離している状態で「再突き抜け」と判定されていました。ユーザーの指摘通り、これは「復帰」というよりは「反発後の上昇局面」と捉えるのが自然であり、買い2の意図とずれています。
- **改善提案:**
    1.  **買い2の「乖離率しきい値」を厳格化:** `0.0 <= diff_rate <= 5.0` を `0.0 <= diff_rate <= 2.5` に修正します。これにより、長期線から大きく離れた位置での再突き抜けを排除します。
    2.  **買い2に「短期線（25日線）の位置」を追加:** 短期線が長期線より上にあること (`short_ma_today > long_ma_today`) を条件に追加します。
    3.  **買い2に「株価が短期線から離れすぎていないこと」を追加:** 株価が短期線から大きく上に乖離していないこと (`((price_today - short_ma_today) / short_ma_today) * 100 <= 3.0`) を条件に追加し、より「復帰直後」の局面を捉えます。

#### 📁 事例：6552.T (ＧａｍｅＷｉｔｈ) - 2026-07-07 (中期(25/75))
- **現在の出力:** 買い1
- **ユーザーのフィードバック:** 「株価が75日線を下回っている期間が短く、買い1と判定できない」
- **分析:**
    *   買い1は「新規買い初動」として、長期にわたる下降・調整からの転換を意図しています。現在の条件 `below_count_20d >= 12` は「過去20日間のうち12日以上」と、約6割の期間下回っていれば良いという条件でした。これがユーザーの「期間が短い」という感覚と乖離していました。
- **改善提案:**
    1.  **買い1の「下抜け期間」条件を厳格化:** `below_count_20d >= 12` を `below_count_30d >= 18` に修正します。これにより、過去30日間のうち18日以上（約6割）は長期線の下に沈んでいたことを確認し、より明確な調整期間からの回復を狙います。

#### 📁 事例：1905.T (テノックス) - 2026-07-09 (中期(25/75))
- **現在の出力:** 買い3
- **ユーザーのフィードバック:** 「75日線と25日線の乖離が広がり続けており、買い３の局面とは言えない。」
- **分析:**
    *   当時の75日線が下降トレンドに転じており、25日線乖離率もマイナスで、25日線が75日線から下向きに離れていっている状況が示唆されます。このような局面は「押し目」ではなく、下降トレンドへの転換期である可能性が高いです。
- **改善提案:**
    *   上記のテノックス (2026-07-04) および日比谷総合設備 (2026-07-04) と同様、買い3の「長期線の安定的な上昇」条件をより厳しくし、「短期線（25日線）が長期線の上にあること」および「短期線が下向きになっていないこと」を追加することで、下降トレンド中の誤判定を防ぎます。

#### 📁 事例：2760.T (東京エレクトロン　デバイス) - 2026-07-09 (中期(25/75))
- **現在の出力:** 買い3
- **ユーザーのフィードバック:** 「数か月前に25日線が大きく75日線から離れているが、現在下落中の25日線が下向きのままであり、トレンド転換の兆しが見えないため買い3と判断するには早い。」
- **分析:**
    *   75日線は上昇しているものの、25日線が下向きで、75日線から下に乖離している（デッドクロス寸前か後）状況です。このような場合は「押し目」ではなく、トレンドの悪化と判断するのが自然です。ローソク足も陰線でした。
- **改善提案:**
    *   上記の買い3の改善提案（「長期線の安定的な上昇」条件強化、「短期線が長期線の上」と「短期線が下向きでない」の追加）を適用することで、この種の誤判定を排除します。特に、短期線が長期線の下にある場合は、買い3とは見なしません。

#### 📁 事例：3071.T (ストリーム) - 2026-07-09 (短期(5/25))
- **現在の出力:** 期待度3
- **ユーザーのフィードバック:** 「移動平均線が5，25，75すべてで下向きであり、上昇トレンド転換が起こっていると考えられないため、乖離率の条件を満たしていても期待度を下げるべき。」
- **分析:**
    *   短期(5/25)システムでの評価ですが、75日線も下降しており、全体的なトレンドが悪化している状況でした。現在の期待度スコアのロジックでは、個別の条件（出来高急増、相対変化率など）での加点はあっても、複数の移動平均線が下降トレンドにある場合の総合的な減点ロジックが不足していました。
    *   ローソク足の実体率も0%と、反発力が非常に弱いにもかかわらず、減点にはつながっていませんでした。
- **改善提案:**
    1.  **期待度スコアに「総合トレンド悪化」の減点ロジックを追加:**
        *   長期線（中期システムでは75日線、短期システムでは25日線）が下降トレンドの場合 (`long_ma_slope_10d < 0`)、スコアを`-1`点減点。
        *   短期線（中期システムでは25日線、短期システムでは5日線）が長期線より下にある場合 (`short_ma_today < long_ma_today`)、スコアを`-1`点減点。ただし、買い1と買い4は回復局面を狙うため、この条件は除外します。
        *   短期線（中期システムでは25日線、短期システムでは5日線）自体が下降トレンドの場合 (`short_ma_slope_3d < 0`)、スコアを`-1`点減点。ただし、買い1と買い4は回復局面を狙うため、この条件は除外します。
    2.  **買い2にも「反発時の実体が極小」の減点ロジックを追加:** 買い1、買い3、買い4には同様の減点がありますが、買い2にも `candle_body_pct < 0.5` の場合に`-1`点減点する条件を追加します。

### 2. 提案する新たな数式条件とロジック（コードへの反映内容）

`evaluate_logic`関数に以下の変更を適用します。

*   **追加計算:**
    *   `long_ma_slope_5d`: 長期線の直近5日間の傾き。
    *   `short_ma_slope_3d`: 短期線の直近3日間の傾き。
    *   `below_count_30d`: 株価が長期線を過去30日で下回った日数。

*   **買い1：新規買い**
    *   `is_new_crossover` の条件を `below_count_20d >= 12` から **`below_count_30d >= 18`** に変更。

*   **買い2：再突き抜け**
    *   乖離率のしきい値を `0.0 <= diff_rate <= 5.0` から **`0.0 <= diff_rate <= 2.5`** に変更。
    *   **追加条件:** `short_ma_today > long_ma_today` (短期線が長期線の上にあること)。
    *   **追加条件:** `((price_today - short_ma_today) / short_ma_today) * 100 <= 3.0` (株価が短期線から3%以内に収まっていること)。

*   **買い3：押し目反発 / 待ち伏せ**
    *   `is_long_ma_rising_strong` を **`(long_ma_slope_15d > 0 and long_ma_slope_5d > 0 and long_ma_today > long_ma_yesterday)`** に強化。
    *   **追加条件:** `short_ma_today > long_ma_today` (短期線が長期線の上にあること)。
    *   **追加条件:** `short_ma_slope_3d >= 0` (短期線が下降トレンドにないこと)。
    *   通常の買い3 (`BUY3`) の乖離率しきい値を `0.0 < diff_rate <= 3.5` から **`0.0 < diff_rate <= 2.0`** に変更。

*   **期待度スコア**
    *   **買い2 (`BUY2`) の減点追加:** `candle_body_pct < 0.5` の場合に `-1` 点減点。
    *   **全体トレンド悪化による減点追加:**
        *   **`long_ma_slope_10d < 0`** の場合、スコアを `-1` 点減点。
        *   **`category not in ["BUY1", "BUY4"] and short_ma_today < long_ma_today`** の場合、スコアを `-1` 点減点。
        *   **`category not in ["BUY1", "BUY4"] and short_ma_slope_3d < 0`** の場合、スコアを `-1` 点減点。

これらの改善により、ダマシを減らしつつ、グランビル法則の意図に合致した優良銘柄を抽出できる精度が向上すると考えられます。
